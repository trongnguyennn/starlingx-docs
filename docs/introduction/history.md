# History of StarlingX

StarlingX originated as a community-driven project to address the needs of edge and telco operators for an integrated platform combining virtualization, storage, and cloud-native orchestration. Over time, it evolved with contributions from industry partners to support hardened production features and lifecycle tooling.

## Evolution highlights

- Early focus on integrating upstream OpenStack components for edge use-cases.
- Introduction of Kubernetes integration and Ceph for storage resilience.
- Continued enhancements targeted at automated deployment and Day-2 operations.
